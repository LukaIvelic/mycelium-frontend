# Mycelium Console

This project uses Next.js with a required app `basePath` of `/console` and Auth.js (`next-auth` v5 beta) for Google sign-in.

## What Broke Initially

The initial Auth.js setup had two classes of issues:

1. Route resolution mismatch
- App routes run under `/console` because of `basePath`.
- Auth.js client calls were still attempting `/api/auth/*`.
- Result: `404` on endpoints like `/api/auth/providers` and `/api/auth/error`.

2. OAuth redirect mismatch with Google
- Google OAuth responded with `Error 400: redirect_uri_mismatch`.
- The redirect URI generated by the app did not exactly match allowed URIs in Google Cloud OAuth settings.

## Root Cause

`basePath` in Next.js and Auth.js client/server URL generation need to be explicitly aligned.
If they drift, either Auth endpoints 404, or OAuth callback URIs mismatch.

## Fixes Applied

### 1) Keep Next.js mounted under `/console`
In `next.config.ts`:
- `basePath: "/console"` remains enabled.

### 2) Add auth path alias for generated `/api/auth/*` URLs
In `next.config.ts` redirects:
- `/api/auth/:path*` -> `/console/api/auth/:path*`.

This allows Auth.js-generated `/api/auth/*` links to still resolve in a `/console` deployment.

### 3) Configure Auth.js server route base
In `src/auth.ts`:
- `basePath: "/api/auth"`
- Google provider reads credentials from env vars:
	- `AUTH_GOOGLE_ID`
	- `AUTH_GOOGLE_SECRET`

### 4) Configure Auth.js React client base path
In `src/app/providers.tsx`:
- Wrap app with `SessionProvider basePath="/console/api/auth"`.

In `src/app/layout.tsx`:
- Wrap app children with `Providers`.

### 5) Use proper sign-in flow
In `src/app/auth/_page.config.ts`:
- Google button uses `signIn("google")` from `next-auth/react`.

## Environment Variables

Required env vars:

```env
AUTH_SECRET=...
NEXTAUTH_URL=http://localhost:3001/api/auth
AUTH_URL=http://localhost:3001/api/auth
AUTH_GOOGLE_ID=...
AUTH_GOOGLE_SECRET=...
```

For production, replace host:

```env
NEXTAUTH_URL=https://www.myceliums.dev/api/auth
AUTH_URL=https://www.myceliums.dev/api/auth
```

## Google OAuth Console Setup

For local + production, OAuth client should include:

### Authorized JavaScript origins
- `http://localhost:3001`
- `https://www.myceliums.dev`

### Authorized redirect URIs
- `http://localhost:3001/api/auth/callback/google`
- `https://www.myceliums.dev/api/auth/callback/google`

Important: URI matching is exact. Any mismatch in protocol/domain/path causes `redirect_uri_mismatch`.

## Run Locally

```bash
npm install
npm run dev
```

App URL:
- `http://localhost:3001/console/auth`

## Troubleshooting Checklist

If Google login fails:

1. Confirm `AUTH_URL` and `NEXTAUTH_URL` match your current environment domain.
2. Confirm Google OAuth redirect URI includes `/api/auth/callback/google` exactly.
3. Confirm deployed env vars are actually updated (not only local `.env`).
4. Wait a few minutes after editing Google OAuth settings.
5. Re-deploy/restart after env changes.
